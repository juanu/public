Testing and Playing Around

So, this is not my first attempt to make a website/blog/some presence in the web, but I think so far seems to be the one that I have developed more quickly into something useful.

I thought about using Wordpress or a similar platform, but I always thought that was a little too much for my purposes. Then I noticed that a lot of people were using static website generators, such as Jekyll (Based on Ruby) or Hyde (Based on Python). As I’m not expert in Ruby, I was looking into Hyde (not that I’m an expert on Python either, but I have a little more experience), but I was a little dissapointed with the lack of documentation. Then I found this: Octopress, which is a framework based on Jekyll, which already provides the necessary basic structure (HTML, CSS, Javascript) files to start with a website. I decided to give a try, so I could start building a site/blog, and learn about the details during the process.

Some of the reasons that made static site generators attractive were:

  • “Simplicity”: I put this in quotes, because it took me a while to figure out the programs, dependencies, and things that I needed to be able to run. But, once that everything was running, it just takes a few minutes of buliding and the website is ready!

  • More accessible: I don’t know the exact word to refer to this, but for me, is much easier to write a quick blog post using Markdown (a lightweight markup language) in a text editor, than going to Wordpress, login in, start writing, etc. I see this as a process similar to using Latex, where you are not worried about the design of the text (like in Word), but more about the content itself. Of course, I’m saying this right now, but let’s see if this translate into more writing!

  • Learning: I think it was time to start learning some serious HTML, CSS and even Javascript!

So, after a few days, the experience has been interesting, and I’ll see how this works in the future. Feel free to ask any questions or comments!
